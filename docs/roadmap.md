# 로드맵

## 목표

LogLens의 목표는 단순 파일 뷰어가 아니라, 로컬에서 바로 실행 가능한 로그 분석 및 디버깅 워크벤치가 되는 것입니다.

- 로그를 빠르게 열고 검색한다.
- 여러 로그 이벤트 사이의 관계를 따라간다.
- 요청 흐름을 trace/span 관점에서 재구성한다.
- 장애 분석과 디버깅에 필요한 컨텍스트를 한 화면에서 탐색한다.

## 현재 기반

2026-03-08 기준 현재 구현은 다음 수준까지 와 있습니다.

- 단일 `.log`/`.txt` 파일 선택
- 줄 단위 로그 테이블 조회
- 검색어 기반 필터링
- 일부 공통 타임스탬프 형식 기반 시간대 분포 차트
- 선택 파일 단위 Tauri 파일 접근 허용
- JS audit 취약점 정리와 주요 의존성 최신화

## 바로 다음 할 일

### 1. 로그 모델 정의

- raw line 중심 상태에서 벗어나 `timestamp`, `level`, `service`, `message`, `traceId`, `spanId`, `parentSpanId`, `fields` 같은 공통 필드를 정의한다.
- 파서 실패 라인도 보존할 수 있게 원본 라인과 파싱 결과를 함께 저장한다.
- 파서 계층을 UI 컴포넌트 밖으로 분리한다.

### 2. 분석 UX 확장

- 시간대 차트 외에 레벨 분포, 서비스별 빈도, 주요 키워드 집계를 추가한다.
- 검색어 외에 structured filter UI를 만든다.
- 검색 결과와 차트가 서로 drill-down 되도록 연결한다.

### 3. 관계 추적

- trace/span ID, request ID, correlation ID를 우선 지원 대상으로 정한다.
- 동일 ID를 가진 로그를 타임라인으로 묶는다.
- parent/child span 관계를 시각화하는 기본 트리 또는 간트형 뷰를 만든다.

### 4. 대용량 파일 대응

- 파일 전체를 한 번에 메모리에 올리는 구조를 스트리밍/청크 기반으로 바꾼다.
- 테이블에 가상 스크롤을 도입한다.
- 여러 파일을 세션 단위로 관리할 수 있게 한다.

### 5. 신뢰성

- 샘플 로그 fixture를 추가한다.
- 파서 단위 테스트와 UI 스모크 테스트를 만든다.
- 지원하는 타임스탬프 형식과 파싱 실패 규칙을 문서화한다.

## 중기 방향

- 다중 파일 correlation
- 저장된 분석 세션
- 북마크/메모/주석
- 특정 에러 패턴 탐지
- OpenTelemetry 스타일 trace 데이터와의 연결 가능성 검토

## 장기 방향

- 로컬 인덱싱
- 빠른 전문 검색
- 플러그인형 파서
- 외부 로그 소스 import
- 팀 공유용 export/report
